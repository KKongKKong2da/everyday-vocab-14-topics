# Everyday Vocab: 14 Topics

> **알고 있었지만 바로 떠오르지 않는 영어 단어를, 생활 주제별로 다시 꺼내는 단어장**

**일상 영단어 14개 주제**는 사람, 집, 음식, 동네, 교통, 직장, 병원, 취미처럼 생활에서 자주 만나는 어휘를 주제별 카드로 학습하는 웹앱입니다.

영어를 처음부터 설명하는 입문 교재라기보다, 예전에 배웠지만 기억이 흐릿해진 단어를 빠르게 확인하고 반복하기 위해 만들었습니다. 단어만 외우는 대신 **뜻·발음·예문을 한 카드에서 함께 보고**, 기억 정도를 직접 표시하며 복습할 수 있습니다.

[**웹앱에서 학습하기**](https://kkongkkong2da.github.io/everyday-vocab-14-topics/)

> GitHub Pages 배포가 완료된 뒤 위 주소에서 바로 사용할 수 있습니다.

## 이런 학습자에게 잘 맞습니다

- 쉬운 단어도 막상 말하려면 바로 떠오르지 않는 사람
- 일상생활에 가까운 어휘부터 차근차근 다시 정리하고 싶은 사람
- 긴 단어 목록보다 주제별로 묶인 단어장을 선호하는 사람
- 단어의 뜻뿐 아니라 발음과 실제 문장까지 함께 확인하고 싶은 사람
- 출퇴근이나 쉬는 시간에 짧게 여러 번 복습하고 싶은 사람

## 이 단어장의 특징

### 1. 일상에 가까운 순서로 구성했습니다

학습은 **나와 가족 → 집 → 음식과 쇼핑 → 시간과 날씨 → 동네 → 이동 → 직장 → 병원 → 취미와 문화** 순으로 이어집니다.

국가명처럼 한꺼번에 길게 이어지는 목록은 본문 학습에서 분리해 **부록 A. 국가·수도 색인**으로 옮겼습니다. 기본 학습 흐름은 생활 어휘에 집중하고, 국가명은 필요할 때 검색하거나 부록에서 따로 볼 수 있습니다.

### 2. 한 카드에서 단어와 문장을 함께 봅니다

각 카드에는 다음 정보가 들어 있습니다.

- 영어 표제어
- 품사와 한국어 뜻
- 영어 발음기호와 한국어 발음 안내
- 의미에 맞춘 영어 예문
- 자연스러운 한국어 해석
- 표제어와 강세를 알아보기 쉽게 표시한 서식

뜻만 확인하고 넘기는 방식보다, 단어가 문장 속에서 어떻게 쓰이는지 함께 익힐 수 있도록 구성했습니다.

### 3. 기억 상태를 직접 표시할 수 있습니다

각 단어를 다음 세 단계로 구분할 수 있습니다.

- **낯섦** — 아직 익숙하지 않은 단어
- **익힘** — 뜻은 알지만 더 반복해야 하는 단어
- **익숙** — 바로 이해하거나 사용할 수 있는 단어

학습 상태는 브라우저에 자동으로 저장됩니다. 다음에 다시 접속해도 이어서 학습할 수 있고, **저장 코드 또는 QR 코드**를 이용해 다른 브라우저나 기기로 옮길 수도 있습니다. GitHub Pages에서 QR을 스캔하면 웹앱이 열리면서 학습 기록이 자동으로 적용됩니다.

### 4. 보고, 듣고, 가리고, 다시 확인합니다

- 단어와 예문의 영어 음성 재생
- 보통 속도와 느린 속도 전환
- 뜻 숨기기
- 순차 학습과 랜덤 학습
- 영어·한국어 뜻·주제 통합 검색
- 14개 대분류와 세부 주제 필터
- 현재 필터 위치와 전체 고정 단어 번호 표시
- 저장 코드·QR 코드를 이용한 기기 간 학습 기록 이동
- PC와 모바일 화면 대응

별도의 회원가입이나 서버가 필요하지 않습니다.

## 추천 학습 방법

### 하루 5분 복습

1. 주제 하나를 선택합니다.
2. 카드 10~20개를 순서대로 봅니다.
3. 뜻을 가린 채 단어의 의미를 먼저 떠올립니다.
4. 발음과 예문을 들어 봅니다.
5. 기억 정도를 `낯섦 / 익힘 / 익숙`으로 표시합니다.

### 기억이 흐릿한 단어만 다시 보기

처음에는 빠르게 전체를 훑고, `낯섦`과 `익힘`으로 표시한 단어를 중심으로 반복하는 방식이 효율적입니다. 완벽하게 외우고 다음으로 넘어가기보다, 여러 번 다시 만나는 것을 목표로 합니다.

### 말하기 연습으로 확장하기

예문을 들은 뒤 화면을 보지 않고 한 번 따라 말해 보세요. 익숙해지면 예문의 인물, 장소, 시간만 바꿔 자신의 문장으로 만들어 보는 것도 좋습니다.

## 학습 기록 저장

학습 상태는 현재 브라우저의 `localStorage`에 저장됩니다.

- 같은 브라우저에서는 자동으로 이어집니다.
- 브라우저 데이터를 삭제하면 기록도 사라질 수 있습니다.
- 다른 기기로 옮길 때는 앱의 **저장 / 불러오기** 기능을 사용하세요.
- `저장 → QR`을 누르면 다른 기기의 카메라로 스캔할 수 있습니다.
- GitHub Pages에서 만든 QR은 앱 주소와 학습 상태를 함께 담습니다. 스캔하면 앱이 열리고 기록이 자동으로 적용됩니다.
- 내려받은 `index.html`을 직접 연 경우 QR에는 공개 웹주소가 아니라 학습 상태 코드만 담깁니다.

공유 주소의 `#s=...` 부분은 브라우저에서만 읽고 적용한 뒤 주소창에서 제거합니다. 개인 학습 기록은 별도의 서버로 전송하지 않습니다.

## 비개발자가 AI와 함께 만든 과정

이 프로젝트는 개발 경험이 없는 제작자가 **필요한 학습 방식과 데이터 기준을 직접 정의하고, AI를 개발 파트너처럼 활용해 완성한 웹앱**입니다.

처음부터 완성된 코드를 한 번에 생성한 것은 아닙니다. 실제 사용 중 발견한 문제를 하나씩 설명하고, AI가 제안한 구현을 확인한 뒤 다시 수정하는 과정을 반복했습니다. 개발자 관점에서 보면 이 프로젝트는 단순한 코드 생성보다 **요구사항 정의, 데이터 설계, 반복 검수와 개선**에 더 가까운 작업입니다.

### 1. 코드보다 먼저 학습 경험을 정의했습니다

먼저 어떤 기능을 만들지보다, 학습자가 어떻게 사용할지를 정했습니다.

- 한 화면에서 단어·뜻·발음·예문을 함께 볼 것
- 뜻을 가리고 먼저 떠올릴 수 있을 것
- `낯섦 / 익힘 / 익숙`으로 기억 상태를 남길 것
- 일상에 가까운 주제부터 순서대로 학습할 것
- 국가명처럼 긴 목록은 본문에서 분리할 것
- 설치나 회원가입 없이 바로 사용할 수 있을 것

이 기준이 웹앱의 화면 구조, 데이터 형식, 필터 방식과 저장 기능을 결정했습니다.

### 2. 영단어장을 먼저 구조화된 데이터로 만들었습니다

웹 화면보다 먼저 3,557개의 단어와 3,898개의 예문을 표 형태로 정리했습니다. 각 항목에는 표제어, 품사, 뜻, IPA, 한국어 발음, 예문, 해석, 대분류와 소분류를 같은 규칙으로 기록했습니다.

AI는 다음과 같은 반복 작업을 보조했습니다.

- 품사와 뜻의 누락 확인
- 뜻의 수에 맞는 예문 작성과 보완
- 예문 속 표제어 강조
- 한국어 발음의 강세 표시
- 반복되는 문장 패턴 탐지와 재작성
- 분류명과 학습 순서 정리
- XLSX·CSV·Markdown 간 내용 비교

다만 어떤 단어를 넣을지, 어떤 표현이 학습에 필요한지, 예문이 자연스러운지에 대한 최종 판단은 제작자가 직접 내렸습니다.

### 3. 별도 서버가 없는 단일 HTML 웹앱으로 구성했습니다

웹앱은 프레임워크나 백엔드 없이 **HTML, CSS, JavaScript만으로 동작하는 정적 웹앱**입니다.

- 영단어 데이터는 `index.html` 안에 포함
- 검색과 주제 필터는 브라우저에서 처리
- 영어 음성은 Web Speech API의 `speechSynthesis` 사용
- 학습 상태는 브라우저의 `localStorage`에 저장
- 학습 상태 공유 코드는 희소·고정 길이 중 더 짧은 형식을 선택하고 CRC32로 오류를 검증
- QR 코드는 브라우저 안에서 SVG로 생성하고, URL fragment(`#s=...`)로 다른 기기에 전달
- GitHub Pages를 통해 정적 사이트로 배포

따라서 별도의 데이터베이스, 로그인 서버, 빌드 과정 없이 `index.html` 하나만으로도 실행할 수 있습니다. 네트워크 연결 없이 파일을 직접 열어 사용하는 것도 가능합니다. 다만 음성 기능은 브라우저와 운영체제가 제공하는 음성 엔진의 영향을 받습니다.

QR 공유도 서버를 거치지 않습니다. 앱이 현재 학습 상태를 짧은 코드로 압축한 뒤 QR 이미지에 넣고, 다른 기기에서 해당 주소를 열면 브라우저가 코드를 해석해 `localStorage`에 저장합니다. QR 생성기는 단일 HTML 안에 포함되어 있어 외부 라이브러리를 내려받지 않습니다.

### 4. AI와의 개발은 짧은 반복 단위로 진행했습니다

작업은 대체로 다음 순서로 반복했습니다.

1. 필요한 기능이나 발견한 문제를 구체적으로 설명합니다.
2. AI가 HTML·CSS·JavaScript 또는 데이터 수정안을 만듭니다.
3. 실제 브라우저와 파일에서 결과를 확인합니다.
4. 어색한 화면, 잘못된 데이터, 예외 상황을 다시 전달합니다.
5. 수정본을 비교하고 문제가 없어질 때까지 반복합니다.

예를 들어 단순히 “검색 기능을 만들어 줘”라고 요청하기보다, **영어 표제어·한국어 뜻·주제명을 함께 검색하고, 본문에서 분리한 국가명도 검색 결과에는 포함해야 한다**는 식으로 동작을 구체화했습니다.

### 5. 생성보다 검수에 더 많은 비중을 두었습니다

AI가 만든 결과는 그대로 사용하지 않고, 다음 항목을 반복해서 검사했습니다.

- 카드와 예문의 개수가 원본과 일치하는지
- 누락되거나 중복된 항목이 없는지
- XLSX·CSV·Markdown·HTML의 내용이 같은지
- 분류 이동 후 모든 단어가 올바른 위치에 있는지
- 굵은 강세와 예문 표제어 서식이 유지되는지
- JavaScript 오류 없이 웹앱이 실행되는지
- 모바일에서도 필터와 카드가 정상적으로 보이는지

이 과정에서 AI는 구현과 자동 검사를 도왔고, 제작자는 결과를 직접 읽고 사용하면서 품질 기준을 계속 조정했습니다.

### 6. 이 프로젝트에서 AI가 맡은 역할

AI는 코드를 대신 작성하는 도구이면서 동시에 다음 역할을 했습니다.

- 요구사항을 기능 단위로 나누는 기획 보조
- 대량 텍스트와 데이터의 정리·변환
- HTML·CSS·JavaScript 구현
- 오류 원인 분석과 수정안 제안
- 반복 패턴과 누락 항목 검사
- README와 배포 구조 정리

하지만 프로젝트의 목적, 학습 대상, 분류 원칙, UI에서 무엇을 남기고 뺄지, 어느 결과가 충분히 자연스러운지는 AI가 아니라 제작자가 결정했습니다.

> 이 프로젝트는 “AI가 한 번에 만들어 준 결과물”이라기보다, 비개발자가 자신의 기준을 명확히 설명하고 결과를 반복해서 검수하며 완성한 **AI 보조 개발 사례**입니다.
